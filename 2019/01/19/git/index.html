預備知識

git是啥就不多說了

git有三種主要狀態：committed，modified，和staged

  • committed代表數據已經安全的存入本地數據庫
  • modified代表你對文件進行了更改，而且沒有將更改後的文件committed入數據庫
  • staged代表你已經對更改的文件進行標記進行下一次commit

git project有三個section

如下圖所示：the git directory, the working tree, the staging are

  • git directory是git為你的項目存儲數據的區域，這是git最重要的一部分。從其他計算機克隆倉庫時，拷貝的也是這裡的數據
  • working tree是對項目的某個版本獨立提取出來的內容。也就是你本地的項目，供你修改
  • staging area是一個文件，保存了下次將commit的文件列表信息，一般在git倉庫目錄中

git工作流程：

  • 在working tree（本地）修改文件
  • 暫存文件，將文件快照放入staging area
  • 提交更新。找到暫存區域的文件，將快照永久性存儲到git directory
